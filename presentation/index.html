Every commit in git has a unique hash by which it is referenced.

git log: hash, Author, Date and the state of our Local Repository and the latest local information about remote branches
Note that the actual commit hashes will be different for you.
To compare that commit and the one one before we can do git diff (commit)^!, where the ^! tells git to compare to the commit one before.
OR git diff (from commit) (to commit)

Merging

  • Fast forward merging
  • Divergent branches merging

A merge introduces a merge commit in which the two histories get integrated again.

Rebasing

Another clean way to integrate changes between two branches. Viewed simply, rebasing just changes the point in history (the commit) your branch is based on.
